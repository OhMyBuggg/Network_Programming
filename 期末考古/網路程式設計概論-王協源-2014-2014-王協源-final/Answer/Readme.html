重點就是，所有code都要自己打過一遍進去才會寫（如果時間很趕用抄的也要抄過一遍！），不要只用看的

第一題

  • 直接 make 即可編譯
  • 為符合助教的測試方式，由client程式內自行亂數決定waiting的時間
  • 可將測試的測資以空白或換行隔開寫入檔案後直接pipe給client測試
    • 請參考test檔案, Ex: ./client < test

第二題

  • 這題是直接從上課投影片裡的code抄上去再做修改的，因此比較亂
  • 基本上就是把ping打出來，再用一個for 1~254各跑一次就對了
  • 直接 make 即可編譯，測試方式請參考題目最後兩頁

第三、四題

  • 第三題要改寫課本的 readline 寫出 writeline，實作上更簡單
  • 第四題是第三題的進階版，要將 writeline 改為thread-safe
  • 這邊我直接寫第四題了，所以只有一份code
  • 類似這種的改寫題通常很簡單，code也通常是更少，不要輕易放掉（這題只有54行）
  • 34/test34 內放的是測試的程式，程式與第一題一樣但 write 的部分都改用 writeline
    • 在client中有一個thread，用以測試第三題不須thread-safed時的狀況
    • server為multi-threaded，可測試第四題是否達到thread-safe
    • 直接 make 即可編譯，使用方式與第一題相同
